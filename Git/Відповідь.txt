Первое что приходит на ум это:
При условии что Git уже установлен и настроен на компьтере.
1. Проинициализировать Git - git init
2. Склонировать репозиторий - git clone git@example.com:example/test.git
3. Выбрать место куда будут загружены файлы, если нужно в какую-то определенную папку.
4. Добавить все изменения в Git если не добавлены - git add .
5. Сделать коммит - git commit -m "Название коммита"
6. Запушить коммит в мастер ветку удаленного репозитория - git push origin master

Видел ещё такой способ:
1. Проинициализировать Git - git init
2. Добавить все изменения в Git если не добавлены - git add .
3. Сделать коммит - git commit -m "Название коммита"
4. Связать мой локальный репозиторий с удаленным репозиториаем клиента git remote add origin git@example.com:example/test.git
5. Отправить файлы из моего локального репозитория в удаленный репозиторий клиента git push -u origin master

Так же можно через Pull request:
1. Проинициализировать Git - git init
2. Склонировать репозиторий - git clone git@example.com:example/test.git
3. Выбрать место куда будут загружены файлы, если нужно в какую-то определенную папку.
4. Добавить все изменения в Git если не добавлены - git add .
5. Сделать коммит - git commit -m "Название коммита".
6. Запушить мою ветку в удаленный репозиторий клиента -git push origin my-feature-branch
7. Создать Pull Request:
	- Перейти на сайт GitHub, GitLab или Bitbucket.
	- Создайте Pull Request, выбрав мою новую ветку my-feature-branch как исходную и ветку master клиента как конечное место нахождения файла.
	- Буду ждать пока клиент одобрит изменения
